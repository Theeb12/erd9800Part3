client dir:

package.json: packages used for developing the frontend
/build: autogenerated build files using the yarn build system
/public: assortment of files used by frontend (images, etc.)

/src: source code of the frontend
/src/utility.js: basic functions used for frontend
/src/App.js: basic routing to pages (login, register, etc.)
/src/pages: frontend pages that the 'App.js' routes to
/src/components: code for actual frontend logic (also includes css files for component design)
/src/components/UI: symbols and other images used for frontend design


server dir:

requirements.txt: list of dependencies
utility.py: basic functions used for backend
/database_init: files to initialize database structure and sample data

server.py: main script for backend, calls other components and is run to initialize server
database_reset: function that resets database values
constant.py: list of constants to keep code clean
.env.example: .env example file used for testing

/apis: api implementations
/apis/ "component" : api implementations for a specific component
